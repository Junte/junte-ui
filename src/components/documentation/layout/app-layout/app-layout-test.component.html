Layout

Use stack to layout the views of your interface automatically. Use Layout when you require precise placement of your views.

Overview

    When you design your app’s interface, you position views and other interface elements in your app’s windows and size them appropriately. However, the size and position of those views may need to change at runtime for a variety of reasons:

  1. The user can resize the window containing your views.
  2. Variations in the screen sizes of mobile devices (including differences between portrait and landscape orientations) require different layouts for each device and orientation.
  3. Apps on desktop must adapt to cover different amounts of screen space, ranging from a third of the screen to the entire screen.
  4. Language changes might require size changes for labels and other text-based views.

Requirements

  1. Aside must have drawer with overlay on mobile device
  2. Mobile menu must have 2 levels with icons near menu item
  3. Mobile menu must have compact design
  4. Layout must include parts: header, aside, main, content, body, footer
  5. Header must include logo, menu and userbar
